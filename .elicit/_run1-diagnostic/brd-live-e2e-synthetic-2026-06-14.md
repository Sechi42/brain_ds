## 2. Current State Analysis

### Known Data Assets

**[[Synthetic Orders DB]]** is a SQLite fixture located at `tests/fixtures/synthetic_source.db`. It contains two tables:

- **[[customers]]** — master dimension table (4 columns, ~5 rows). Tracks customer identity, market segment (Enterprise, SMB, Mid Market), and geographic region (LATAM, North America, EMEA).
- **[[orders]]** — central fact table (5 columns, ~5 rows). Records order monetary value, fulfillment status (fulfilled, pending, cancelled), and creation date. Joined to [[customers]] via `customer_id`.

**Known data quality issues in [[Synthetic Orders DB]]:**
- `order_total` stored as REAL — precision risk for monetary calculations.
- `created_at` stored as TEXT (ISO 8601) — type coercion required in query engines.
- Foreign key constraint between [[orders]] and [[customers]] is not enforced at the SQLite DDL layer.
- One customer (Delta Foods, customer_id=4) has no orders — referential integrity gap.
- Full enumerations for `segment`, `region`, and `status` fields are not confirmed.
